# مراقب البطارية

تطبيق سطح مكتب بنظام Windows يعرض معلومات البطارية بشكل لحظي بواجهة عربية (RTL) بسيطة وواضحة. يعرض نسبة الشحن مع مؤشر دائري جذاب، بالإضافة إلى تفاصيل مهمة مثل السعة التصميمية، سعة الشحن الكامل، كيمياء البطارية، مصدر الطاقة، والحالة الحالية للشحن.

## الفكرة
تقديم لوحة معلومات سريعة وسهلة القراءة لمستخدم النظام، بحيث يمكن معرفة حالة البطارية في ثوانٍ دون الحاجة لفتح أدوات النظام المعقدة.

## أهم الميزات
- واجهة عربية كاملة مع اتجاه يمين-إلى-يسار.
- مؤشر دائري لنسبة الشحن مع تأثير ضوئي.
- تحديث تلقائي كل 2 ثانية.
- عرض الحالة الحالية (قيد الشحن/منخفض/حرج/متصل بالكهرباء...).
- عرض الوقت المتبقي التقريبي للشحن/التفريغ.
- عرض بيانات البطارية من WMI:
  - اسم البطارية
  - السعة التصميمية
  - سعة الشحن الكامل
  - كيمياء البطارية
  - حالة البطارية (BatteryHealth)

## كيف يعمل؟
يعتمد التطبيق على مصدرين رئيسيين للبيانات:
- **SystemInformation.PowerStatus** لقراءة النسبة، الحالة، والوقت المتبقي.
- **WMI (Win32_Battery)** لقراءة بيانات البطارية التفصيلية مثل السعة والكيمياء.

كما يستخدم `ArcConverter` لرسم القوس الدائري الذي يمثل نسبة الشحن داخل الواجهة.

## المتطلبات
- نظام Windows (التطبيق مبني بـ WPF).
- .NET Framework 4.7.2 (محدد في ملف المشروع).
- بطارية فعلية على الجهاز لعرض البيانات الفعلية.

## التشغيل محليًا
1. افتح الحل `البطاريه.slnx` في Visual Studio.
2. نفّذ Build للمشروع.
3. شغّل التطبيق مباشرة من Visual Studio أو من ملف التنفيذ الناتج داخل `bin/Release`.

## هيكل المشروع (مختصر)
- `MainWindow.xaml`: واجهة المستخدم.
- `BatteryInfoViewModel.cs`: منطق قراءة بيانات البطارية وربطها بالواجهة.
- `ArcConverter.cs`: تحويل نسبة الشحن إلى قوس رسومي.

## ملاحظات مهمة
- بعض الأجهزة لا تُرجع كل خصائص WMI، لذا قد تظهر قيم "غير متوفر".
- الوقت المتبقي تقديري ويعتمد على معلومات النظام نفسه.
- على الأجهزة المكتبية بدون بطارية ستظهر رسالة "لا توجد بطارية".
